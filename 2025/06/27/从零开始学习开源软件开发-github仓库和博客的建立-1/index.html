从零开始学习开源软件开发——-github仓库和博客的建立

一. 学习过程

在曾老师和李宁老师，窦茂川导师，扎西尖措导师的指导下，我开始学习开源软件开发

（以下内容中斜体为遇到的问题，解决方法是加粗体）

1.准备工作

​ 一开始，扎西尖措导师在课堂上带领我们注册github，很多同学因为没有梯子没办法登录，于是我帮助了一些同学下载VPN，这样子能够很方便快速地进入github和chatgpt等网站。然后我们按部就班地完成node.js的安装和git的安装。

​ 遇到问题：

在下载node.js过程中误入了一个高仿的网站，让我以为是官网，结果进去下载完发现是什么p2p加速器下载，赶紧关掉

2.安装配置hexo

​ 我们先在文件夹中创建了博客的目录，接着安装hexo和进行初始化配置。在初始化过程中遇到问题：没有得到管理员的允许去修改d盘的空间导致失败 。解决方案：在cmd中用管理员身份再运行一遍。这些完成之后，我们测试了最初始的博客页面。来到配置文件_config.yml，修改了博客的标题，副标题和作者名称。来到了最重要的一步——选择主题。

​ 最开始我在hexo官网上选择了一个keep moving的主题，但是下载成功之后发现他需要下载很多插件，而且里面文件的格式也要按照他规定的格式，否则一致出错，于是去选择导师推荐的主题next。但是先是提示我网络连接失败，我试了好多次才成功下载下来，下载之后修改配置文件里面的主题。正当我以为我已经完成的时候打开我的博客确发现是一片空白，网页什么都没有，最开始我以为我那个主题名字打错了，回去检查发现没问题。自己捣鼓了一下搞不明白之后，我问了身边的同学，他们页碰到了这个情况，接着我加了扎西尖措导师的微信，请教之后觉得应该是我的文件没下好。所以我删除原来的主题再重新下载一遍就解决了。

3.撰写，本地预览

​ 我随便创建了一篇新文章，因为我以前自己学习过markdown语法，所以学和写起来很简单也很快发布了自己的第一篇文字虽然是乱打的

4.将博客部署到 GitHub 最复杂

​ 我们先启动了本地的服务器在github中建立了自己的一个个人仓库KevinRunzhi.github.io，接下来就到配置ssh秘钥。我打开git bash 后分别把git config —global user.name “KevinRunzhi” git config —global user.email “1434238938@qq.com” ssh-keygen -t rsa -C “1434238938@qq.com” 复制了回车，结果一直不成功，后面才知道要一次性复制进去再回车，然后跟着PPT把里面给出来的所有内容复制出来粘贴到网站上面，总是提示格式不对。我先把内容复制到豆包上面，让他告诉我该怎么去解决，但是他给出来的解决方案就是说我的格式是错的，让我重新生成。可是重新生成出来的东西也是几乎差不多的，复制进去还不行。自己一直问ai都没搞明白，直到我问一个同学才明白：应该去找已经生成的一个文件，然后进去把生成的秘钥复制出来，最后终于做完了。

2.感想感悟

​ 通过上开源软件开发这个课，让我初步接触到了开源开发，有了自己的github和自己的博客，也增进了自己解决问题的能力。我现在觉得碰到问题之后第一步是自己思考摸索（解决20%），第二步是去询问ai并跟着尝试解决（解决40%），第三步是问同学和老师解决问题。

​ 衷心感谢在我github仓库和博客的建立过程中帮助我的导师和同学们！
